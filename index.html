Capabilities

Everything you need to
assess quantum readiness

From a single domain to an entire subdomain portfolio — the scanner handles discovery, analysis, and reporting in one command.

🔍
TLS Handshake Analysis
Deep inspection of TLS version, cipher suite, key exchange algorithm, and full certificate chain — passive and read-only, no authentication required.
🌐
Subdomain Discovery
Integrates with subfinder to enumerate active subdomains before scanning. Uses -nW -active to filter wildcard and dead subdomains.
📋
CBOM JSON Output
Every scan writes a machine-readable Cryptographic Bill of Materials to output/ — ready for ingestion into SIEMs, CI pipelines, or audit workflows.
📊
Interactive HTML Report
Self-contained dashboard with stats bar, progress bar, filter buttons, and per-asset cards. Subdomains are grouped under their parent domain with an expand button.
🖥️
Web GUI
Browser-based frontend (app.py) — enter hosts, pick options, and watch the scan stream live. No CLI knowledge required. Past reports are one click away.
🛠️
Remediation Guidance
Every asset card includes copy-paste remediation steps for Nginx, Apache, and HAProxy — from enabling X25519Kyber768 to planning ML-DSA certificate migration.
Analysis

What gets inspected

For each host, the scanner analyses eight cryptographic components and classifies each against post-quantum threat models.

Component What is checked Quantum threat
TLS version 1.0 / 1.1 / 1.2 / 1.3 legacy weaknesses
Key exchange X25519, ECDH, X25519Kyber768, ML-KEM Shor's algorithm
Bulk cipher AES-128 / AES-256 / ChaCha20 / 3DES Grover's algorithm AES-256 safe
Hash / MAC SHA-256 / SHA-384 / SHA-512 SHA-384+ safe
Certificate key RSA / ECDSA / ML-DSA Shor's algorithm
Certificate chain Depth, issuer, signing algorithm entire chain must use PQC
Certificate expiry Days remaining warning <30 days / error if expired
Setup

Get running in minutes

Step 1
Clone the repo
git clone https://github.com/sankalp-rajeev/quantumCheck
cd quantumCheck
Step 2
Install Python dependency
pip install -r requirements.txt
Only requires Flask. No other packages beyond the standard library.
Step 3 (optional)
Install subfinder for subdomain discovery
# Linux x86_64
curl -sL https://github.com/projectdiscovery/subfinder/\
releases/download/v2.13.0/\
subfinder_2.13.0_linux_amd64.zip \
  -o /tmp/sf.zip
unzip -o /tmp/sf.zip subfinder -d bin/
chmod +x bin/subfinder
Step 4
Launch the web GUI
python3 app.py
# Open http://localhost:5000
Or use the CLI directly
CLI quick reference
# Single host
./scan.sh google.com

# Custom port
./scan.sh api.bank.com:8443

# Multiple hosts + HTML report
./scan.sh google.com cloudflare.com \
  --report

# Subdomain discovery
./scan.sh example.com \
  --subdomains --report

# From a file
./scan.sh -f domains.txt --report

# JSON only (for scripting)
./scan.sh example.com --json \
  > cbom.json

System requirements: openssl 1.1.1+, python3 3.8+, coreutils (timeout). No root access needed. Works on any Linux/macOS machine with OpenSSL installed.

Output

Three output modes

Every scan produces all three — pick what fits your workflow.

💻
Colour terminal report
Three-phase output: a live progress line per host → summary table with label counts → full per-host detail (TLS, quantum analysis, certificate, recommendations). ANSI-coloured, ASCII art banner.
📄
CBOM JSON
Machine-readable asset inventory written to output/cbom_YYYYMMDD_HHMMSS.json. Includes full connection details, key exchange classification, certificate chain, quantum assessment, vulnerabilities list, and remediation steps.
🌐
Interactive HTML dashboard
Self-contained, no internet required. Stats bar, progress bar, label filters, full asset cards with dot indicators. Subdomains grouped under parent domain with a collapsible "View N subdomains" drawer showing compact per-subdomain rows.
